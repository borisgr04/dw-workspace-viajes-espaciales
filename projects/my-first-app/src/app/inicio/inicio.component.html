Implementación de Plataforma de Vuelos espaciales
Desarrollado con Angular y utilizando
  • Angular Library: Para creación de módulos publicables en npm y reutilizables en otros proyectos angular
  • Angular Elements: Para creación de elementos publicables en npm y reutilizables en cualquier proyecto web (javascript, react.js, vue.js, etc)
  • Angular PWA: Para creación de página progresiva.
  • RxJs: Para manejo de programación reactiva - manejo de observables
  • Comunicación entre componentes:
  • Guards: Para proteger rutas de acuerdo a roles
  • Interceptores: Para simular el envío del token

Funcionalidades implementadas
Implementación parcial de funcionalidades.
  • Menú: Barra de menú superior en la cual se muestran las diferentes opciones que se habilitación o deshabilitan de acuerdo al rol:
    Rol piloto, solo podrá regitrar alquileres.
    Rol admin, solo podra consultar usuarios,
    Rol pasajero, no se alcanzo a implementar alguna consulta. solo puede cerrar sesión (logout)
  • Login: Simulación del inicio de sesión con Api simulada en memoria
  • Inicio: Presenta esta información
  • Consultar usuarios: Muestra la consulta de usuarios al administrador.
  • Registrar alquiler de aeronave: Permite por medio de un formulario reactivo registrar datos, comunicarse con el componente de consulta de usuarios para que solo permita seleccionar usuarios tipo pasajeros.

    Tambien tiene instalado un componente desplegado en Npm llamado dw-toolbar (que contiene los botones guardar y cancelar). dw-toolbar

    Este componente se implemento como una libreria angular y al estar desplegado en npm puede ser instalado en cualquier proyecto angular.

    Este componente se encapsulo dentro del componente dw-ng-toolbar el cual es Angular Elements, que permitió ser públicado como Componente Web que puede ser instalado en cualqueir aplicación web basada en javascript.

    En ambos estilos library y elements tiene publico los manejadores para los eventos de los botones que tiene implementado.

    Al registrar el alquier se manda un mensaje de satisfacción, pero no se almacena dicha información.

    El sistema no permite agregar más de una vez a un pasajero dentro del mismo alquiler de aeronave.

  • Logout: Cierra sesión y por medio del control de un observable permite actualizar las opciones del menú.
